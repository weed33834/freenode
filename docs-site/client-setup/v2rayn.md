# v2rayN 配置教程（Windows）

v2rayN 是 Windows 平台上最受欢迎的 V2Ray / Xray 图形化客户端之一，支持 VMess、VLESS、Trojan、Shadowsocks 等多种协议，适合使用 FreeNode 的 V2Ray / Base64 订阅。

## 准备工作

1. 访问 [v2rayN 官方 Releases](https://github.com/2dust/v2rayN/releases) 下载最新版。
2. 解压到本地目录（建议路径不含中文或特殊字符）。
3. 运行 `v2rayN.exe`。若 Windows 提示 SmartScreen，点击 **更多信息** → **仍要运行**。
4. 首次启动建议以管理员身份运行，以便自动配置系统代理。

## 配置步骤

### 方法一：通过订阅 URL 导入（推荐）

1. 在 v2rayN 主窗口，点击顶部菜单 **服务器** → **从剪贴板导入批量 URL** 或 **订阅分组** → **添加订阅**。
2. 如果使用订阅分组，在 **订阅设置** 中粘贴 FreeNode 的 V2Ray 订阅地址：
   ```
   {{github_url}}/nodes/v2ray.txt
   ```
3. 设置备注名为 `FreeNode`，点击 **确定** 保存。
4. 在主界面右键点击该订阅分组，选择 **更新订阅**。
5. 更新完成后，节点列表会出现在主窗口中。
6. 右键点击一个节点，选择 **设为活动服务器**。
7. 点击底部状态栏的 **系统代理** 按钮，选择 **自动配置系统代理** 或 **自动配置系统代理和不改变路由**。

### 方法二：通过 Base64 链接导入

1. 打开浏览器，访问 `{{github_url}}/nodes/v2ray.txt`。
2. 全选页面内容并复制（即 Base64 编码的节点链接）。
3. 在 v2rayN 中点击 **服务器** → **从剪贴板导入批量 URL**。
4. 客户端会自动解析并导入所有节点。

## 测试连接

1. 选中一个节点后，点击主界面右上角的 **测试服务器延迟**（Ctrl+T）。
2. 绿色数字表示延迟较低，可优先使用。
3. 若某个节点 timeout，换其他节点再试。

## 常见错误排查

| 问题 | 可能原因 | 解决方法 |
|------|----------|----------|
| 更新订阅失败 | GitHub Raw 访问受限 | 开启系统代理后重试，或改用镜像地址 |
| 导入后节点显示为 0 | 订阅内容不是标准 V2Ray URL / Base64 | 确认使用的是 `v2ray.txt`，而非 Clash YAML |
| 所有节点延迟 -1 | 节点已失效或本地防火墙拦截 | 更换节点、关闭防火墙测试，或等待次日更新 |
| 系统代理开启后仍无法访问 | 浏览器使用独立代理设置 | 将浏览器代理设置为 **使用系统代理设置** |
| 提示 "Core 启动失败" | Xray 内核被杀毒软件删除 | 将 v2rayN 目录加入杀毒软件白名单，重新下载内核 |

## 安全提示

> ⚠️ 公开节点由第三方维护，运营者可能查看、记录或篡改你的流量。请仅用于学习网络协议与隐私研究，不要在免费代理/节点环境下登录银行、支付、社交等敏感账户。

- 建议启用 v2rayN 的 **路由** 功能，将国内网站直连，避免不必要的流量经过代理。
- 敏感操作请使用可信商业 VPN 或自建节点。
- 不要从不明来源下载 v2rayN 安装包，以免遭遇篡改客户端。
