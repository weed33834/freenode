# v2rayNG 配置教程（Android）

v2rayNG 是 Android 平台上基于 V2Ray / Xray 内核的免费客户端，支持 VLESS、VMess、Trojan、Shadowsocks 等协议，适合导入 FreeNode 的 V2Ray / Base64 订阅。

## 准备工作

1. 从 [v2rayNG GitHub Releases](https://github.com/2dust/v2rayNG/releases) 下载最新 APK。
2. 若无法访问 GitHub，可从 F-Droid 或其他可信镜像源获取。
3. 在 Android 设置中允许 **安装未知来源应用**。
4. 安装完成后打开 v2rayNG，并授予 **VPN 权限**。

## 配置步骤

### 方法一：通过订阅 URL 导入（推荐）

1. 打开 v2rayNG，点击右上角 **⋮** → **设置**。
2. 找到 **订阅设置**，点击右下角 **+** 添加订阅。
3. 在 URL 栏粘贴 FreeNode 的 V2Ray 订阅地址：
   ```
   {{github_url}}/nodes/v2ray.txt
   ```
4. 备注填写 `FreeNode`，点击右上角 **✓** 保存。
5. 返回主界面，点击右上角 **⋮** → **更新订阅**。
6. 订阅更新完成后，所有节点会显示在主界面列表中。
7. 点击选择一个节点，使其右侧出现选中标记。
8. 点击底部 **V** 形连接按钮，允许建立 VPN 连接。

### 方法二：通过剪贴板导入

1. 在手机浏览器中打开 `{{github_url}}/nodes/v2ray.txt`。
2. 长按页面内容，选择 **全选** → **复制**。
3. 回到 v2rayNG 主界面，点击右上角 **⋮** → **从剪贴板导入**。
4. 客户端会自动解析 Base64 中的节点链接。

## 延迟测试与选优

1. 在主界面点击右上角 **⋮** → **测试全部 TCP 延迟**。
2. 延迟较低（如 100-300ms）且显示具体数字的节点通常可用。
3. 显示 `-1` 或 `timeout` 的节点已失效，建议切换到其他节点。

## 常见错误排查

| 问题 | 可能原因 | 解决方法 |
|------|----------|----------|
| 更新订阅失败 | GitHub Raw 被 DNS 污染或网络不通 | 切换 Wi-Fi / 移动数据，或使用镜像地址 |
| 导入后没有节点 | 订阅格式错误 | 确认订阅为 V2Ray / Base64 格式，而非 Clash YAML |
| 所有节点延迟 -1 | 节点已失效 | 等待 FreeNode 次日更新，或切换其他数据源 |
| 点击连接无反应 | VPN 权限未授予 | 在系统设置中重新允许 v2rayNG 建立 VPN |
| 连接后部分 App 无法翻墙 | 路由模式为绕过局域网 | 在设置中将路由改为 **全局** 或自定义规则 |

## 安全提示

> ⚠️ 公开节点由第三方维护，运营者可能查看、记录或篡改你的流量。请仅用于学习网络协议与隐私研究，不要在免费代理/节点环境下登录银行、支付、社交等敏感账户。

- 建议在 v2rayNG 设置中开启 **分应用代理**，仅让需要的应用走代理。
- 不要在公共 Wi-Fi 下长期保持代理开启，避免中间人攻击。
- 敏感操作请使用可信商业 VPN 或自建节点。
